Compile flags for gfortran

Posted: Sun, 29 Apr 2012
  • fortran

When compiling Fortran, you cannot be sure, that gfortran sees the code the same way like you. For instance, when you assign 1. to a real(8) variable and don‘t notice, that 1. is only real(4).

But when compiling with some flags from this blog, gfortran will warn you about such issues. You can find more information about all flags in the man page of gfortran

Standard warnings

  • -Wall enables a predefined set of standard warnings.
  • -pedantic warns about some nonstandard practices.

Warnings, that might show wrong assumptions/code

  • -Wcharacter-truncation warns, when strings might get truncated.
  • -Wconversion-extra warns about even more conversations than enabled with -Wall.
  • -fimplicit-none adds a implicit none to all functions by default. Always declare ALL variables explicit!
  • -Wintrinsic-shadow warns, when an intrinsic function is shadowed by a local variable/function.
  • -Wsurprising warns about code that might indicate an error.

Warnings for slow code

  • -Warray-temporaries warns when temporary arrays are created, so you can maybe optimize them away.

Development checks

Flags in this section adds runtime checks, which make the code slower. So only add them for testing the program and not for a final build.

  • -fcheck=bounds,mem adds checks for IndexErrors and checks for memory allocation at runtime.
  • -fcheck=all adds even more runtime checks, which one: see man page
